# AI_stockScope — 개발 로드맵 & AI 투자 운용 계획

> 2026-07-17 작성. 원칙: **숫자는 코드, AI는 가설·서술만. 채택은 항상 데이터(백테스트/전방 성과)가 결정.**
> 투자 손익의 책임은 사용자 본인에게 있으며, 이 문서는 투자 권유가 아니다.

## Part B. AI를 이용한 투자 운용 계획

### 역할 분담 (불변 원칙)

```
데이터 수집·지표 계산·주문 집행  = 코드 (결정론, 테스트로 검증)
매매 판단                        = 규칙 (evaluate() 순수 함수 — 실시간·백테스트 동일 코드)
규칙의 개선                      = AI(Claude)가 귀인 통계에서 가설 제안
가설의 채택                      = walk-forward 백테스트 게이트 (심판은 항상 데이터)
실계좌 전환·자금 규모            = 인간(사용자)만 결정
```

### Phase 0 — 오프라인 학습 (지금, KIS 불필요) ✅ 진행 중

- **현황**: 사이클 #1 완료 — 기준 전략(MA20/60+RSI+국면 게이트+ATR 손절) in-sample(2019~24) +26.5% / Sharpe 0.58 / MDD -10.7%, 86거래. 가설 1호(손절폭 확대) **데이터로 기각**.
- **계획**: 유니버스 코스피200급으로 확대 → 사이클 2(게이트 반사실 — 114회 차단이 유익했나), 사이클 3(청산 절제 — 데드크로스 청산 +0.24R의 존재 이유), 사이클 4(수급 필터). 사이클당 채택 변경 최대 1~2개.
- **졸업 기준(사전 등록)**: hold-out에서 Sharpe ≥ 0.8, MDD ≥ -15% 이내, 거래 수 ≥ 30(통계적 최소), buy&hold 대비 위험조정수익 우위.

### Phase 1 — 모의투자 전방 시험 (KIS 연결 후, 1~3개월)

- 자동매매 엔진을 모의계좌에서 장중 가동 (KIS_MOCK=true 전용 — 실계좌는 구조적으로 차단됨).
- 야간 복기 루프 자동화(#6): 매일 귀인 → 주 1회만 가설 채택 판정 (과잉 개입 방지).
- **측정 목표**: ① 백테스트-실체결 괴리(슬리피지·미체결·시가 갭) 정량화 ② 시스템 안정성(주문 누락·중복·오류 0건) ③ 규칙 준수율 100%(엔진이 규칙 밖 행동 0회).
- 주간 리포트를 사용자에게 제공 — 성과보다 **괴리와 사고**를 먼저 보고.

### Phase 2 — 실전 전환 판정 (모의 3개월 후)

**사전 등록 기준** (지금 정해두고 사후 변경 금지 — 사후 합리화 방지):
1. 모의 3개월 기대값 > 0 (거래당 평균 R 기준)
2. 모의 MDD가 백테스트 MDD의 1.5배 이내
3. 백테스트 대비 체결 괴리 < 0.3R/거래
4. 시스템 사고(누락·중복 주문) 0건
- **전부 통과** → 사용자 승인 하에 실계좌 소액 검토. **하나라도 미달** → Phase 0로 회귀(전략 재설계), 실전 연기.

### Phase 3 — 실계좌 소액 (통과 시에만)

- 개시 규모: 잃어도 되는 금액만. 거래당 리스크 1% 유지, **일일 -3% / 월 -8% 서킷브레이커**(초과 시 엔진 자동 정지 + 사용자 알림).
- `KIS_ALLOW_REAL_ORDERS=true`는 **사용자가 직접** 설정 (AI/코드가 켜지 않음 — 현재 구조적으로 보장됨).
- 매주 사용자 리뷰 미팅(리포트 기반), 언제든 원버튼 정지.
- 증액은 3개월 단위, Phase 2 기준 재통과 시에만.

### 리스크 고지 (정직한 한계)

- 모의·백테스트 성과는 실전을 보장하지 않는다 (2025~26 hold-out +48%는 강세장 효과 포함).
- 과거 학습의 근본 한계 = 데이터 스누핑. walk-forward·사전 등록 기준·변경 수 제한으로 완화할 뿐 제거 불가.
- 생존 편향(현재 상장 종목 기준), 일봉 손절(장중 급락 미반영), 갭 체결(-1R 설계가 실측 -1.18R) — 리포트 캐비앳에 상시 명시.
- 단일 브로커(KIS)·단일 시장(KR) 의존 — 장애 시 수동 개입 필요.
